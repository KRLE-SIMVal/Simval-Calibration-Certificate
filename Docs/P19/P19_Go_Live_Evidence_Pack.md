# P19 Go-Live Evidence Pack

Status: draft for System Owner and QA/Laboratory review.

This pack defines the retained evidence expected before routine production use.
It does not approve go-live. The production readiness report remains blocked
until the real host, real Entra tenant, backup/restore, reviewer independence,
ValProbe parser validation, retention, and human approval evidence has been
created and reviewed.

## Evidence Principles

- Retain evidence as files, exported records, screenshots, or signed review
  records in a controlled validation package location.
- Every evidence item must include timestamp, person or system creating it,
  software version or commit, and the environment tested.
- Do not store secrets, bearer tokens, private keys, customer data, or raw
  production paths in source control.
- Use production v1 scope only: `SIMVAL_ENABLED_DISCIPLINES=temperature`.
- Treat any failed check as a deviation until reviewed and dispositioned.

## DANAK And Metrology Basis

The go-live evidence is aligned with the local DANAK requirement extracts:

- AB3: laboratory accreditation scope, competence evidence,
  metrological traceability, and quality assurance of results.
- AB11: GUM-aligned uncertainty evaluation, expanded uncertainty reporting,
  CMC floor discipline, and certificate reporting controls.

No calculation, uncertainty, CMC, rounding, certificate rendering, approval, or
audit logic is changed by this pack.

## Required Evidence Map

| Readiness blocker | Required retained evidence | Recommended evidence key |
|---|---|---|
| `runtime_profile_not_production` | Readiness report generated with `SIMVAL_RUNTIME_PROFILE=production` from the approved production host configuration. | Included in report scope |
| `live_entra_verification_missing` | Approved SIMVal Entra tenant/app registration values verified on the production host; successful `POST /auth/entra/session`; successful `GET /me`; retained `user_session_created` audit event. | `live_entra` |
| `tls_host_verification_missing` | Approved host/TLS boundary verification showing users access the application through the controlled HTTPS endpoint, not direct unauthenticated API exposure. | `tls_host` |
| `backup_restore_verification_missing` | SQLite backup evidence with checksum, size, timestamp, integrity check, and restore drill to a separate target path. | `backup_restore` |
| `reviewer_independence_evidence_missing` | End-to-end workflow evidence showing separate operator, technical reviewer, QA approver, and releaser duties; same-user attempts blocked or controlled as deviations. | `reviewer_independence` |
| `valprobe_parser_validation_missing` | Approved parser validation record covering accepted ValProbe workbook variants, malformed/unsafe workbook rejection, raw-file traceability, and any controlled limitations before routine source-data import. | `valprobe_parser_validation` |
| `retention_policy_approval_missing` | Approved retention decision for certificates, raw source files, validation packages, audit events, database backups, and generated artifacts. | `retention_policy` |
| `final_human_approval_missing` | Signed or otherwise controlled go/no-go approval from System Owner and QA/Laboratory reviewer after reviewing the readiness report and evidence pack. | `human_approval` |
| `production_smoke_evidence_missing` | Runtime smoke evidence JSON generated by `generate_production_smoke_evidence.py` for the same software version. | `smoke_evidence` |

## Evidence File Checklist

| Evidence file or record | Minimum content |
|---|---|
| Controlled pilot validation plan | `pilot-validation-plan.json` and Markdown output from `generate_pilot_validation_plan.py` mapping IQ/OQ/PQ pilot activities to retained evidence keys and stop conditions. |
| Runtime profile evidence | JSON output from `generate_runtime_profile_evidence.py` showing production profile, Entra provider, temperature-only scope, and no readiness blockers without exposing local paths or Entra identifiers. |
| ValProbe parser validation evidence | JSON output from `generate_parser_validation_evidence.py` with parser version, controlled fixture manifest, parser test report, controlled fixture execution report, required coverage, checksums, and reviewer approval status. |
| Backup/restore validation evidence | JSON output from `generate_backup_restore_validation_evidence.py` confirming backup and restore integrity checks, matching checksums, retained source-evidence hashes, and reviewer approval status. |
| Reviewer independence evidence | JSON output from `generate_reviewer_independence_evidence.py` confirming four distinct regulated actors, blocked same-user attempts or approved deviation, hashed actor identifiers, workflow evidence checksum, and reviewer approval status. |
| Retention policy evidence | JSON output from `generate_retention_policy_evidence.py` confirming required retention categories, required policy fields, retained source-policy checksum, and reviewer approval status. |
| Human go/no-go approval evidence | JSON output from `generate_human_approval_evidence.py` confirming System Owner and QA/Laboratory approval, evidence-pack review, readiness-report hash reference, software version, and remaining-deviation disposition without exposing reviewer identifiers. |
| Controlled pilot validation package | Standard `validation-package.json`, Markdown report, and reviewer disposition generated by `generate_pilot_validation_package.py` after all required pilot evidence files exist. |
| Validation package | Full regression result, validation report output, test-suite version, commit/version tested, reviewer disposition. |
| Runtime readiness report | JSON output from `generate_production_readiness_report.py` generated on the approved host. |
| Runtime smoke evidence | JSON output from `generate_production_smoke_evidence.py` showing passed `/health`, `/readiness`, `/app`, and `/app/workflow` checks for temperature-only scope. |
| Live Entra evidence | JSON output from `generate_live_entra_evidence.py` confirming the Entra ID Free provider, tenant/client/audience verification, session exchange, `GET /me`, retained `user_session_created` audit event, local role-mapping review, and reviewer approval without exposing tenant or client identifiers. |
| TLS/host evidence | JSON output from `generate_tls_host_evidence.py` confirming HTTPS endpoint verification, approved hostname verification, certificate validity, direct API exposure review, blocked unauthenticated direct API exposure, and reviewer approval without exposing hostnames. |
| Authentication evidence | Entra tenant/client/audience verification without secrets, successful session exchange, local role mapping, active-user review. |
| Reviewer independence evidence | Audit trail showing independent users for regulated workflow transitions and blocked same-user attempts. |
| ValProbe parser validation | Approved fixture set and review record showing supported workbook variants, rejection cases, raw upload evidence retention, parser-version evidence, and unresolved parser limitations. |
| Backup evidence | Backup file metadata, SHA-256 checksum, SQLite integrity result, restore target, restore result. |
| Retention approval | Retention periods and ownership for database, artifacts, raw files, audit trail, validation evidence, and backups. |
| Go/no-go approval | Date, version, readiness-report path, remaining deviations if any, System Owner decision, QA/Laboratory decision. |

## Final Report Command

```powershell

  --release-version <approved pilot version> `
  --output-dir <controlled evidence path>\pilot-validation-plan
```

Expected result before routine production use:

- Process exit code is `0`.
- Report status is `ready_for_go_live_review`.
- `blockers` is empty.
- Controlled pilot validation plan is retained with the validation package.
- Every verified go-live flag has a matching retained evidence reference using
  the evidence keys in this pack.
- Existing evidence-file references are listed in the report manifest with
  SHA-256 and byte size; missing references are listed as unavailable and keep
  the report blocked.
- Runtime smoke evidence is parsed and must pass, match the same software
  version, and confirm temperature-only scope.
- ValProbe parser validation evidence is retained before routine production
  source-data import is allowed.
- Live Entra, TLS/host, backup/restore, reviewer-independence, ValProbe
  parser-validation, retention-policy, and human-approval evidence files must
  contain JSON with `status` set to `passed`.
- `scope.enabled_disciplines` is exactly `["temperature"]`.
- `scope.auth_provider` is `entra_id_free`.
- Report payload does not expose production database or artifact paths.

## Stop Conditions

Production use remains blocked if any item below is true:

- The readiness report exits with code `2`.
- Any evidence reference points to a missing, uncontrolled, or unreviewed file.
- Any verified go-live flag is supplied without its matching evidence reference.
- Any expected evidence file is missing from the report manifest or lacks
  SHA-256 evidence.
- Runtime smoke evidence is missing, malformed, failed, exposes sensitive
  detail, has a mismatched software version, or is not temperature-only.
- Live Entra, TLS/host, backup/restore, reviewer-independence, ValProbe
  parser-validation, retention-policy, or human-approval evidence JSON is
  missing, malformed, or not passed.
- ValProbe parser validation evidence is missing, unreviewed, or does not cover
  the workbook variants intended for routine production use.
- The report scope includes pressure or differential pressure.
- Production authentication is not Microsoft Entra ID Free.
- Reviewer independence could not be demonstrated and no controlled deviation
  has been approved.
- Backup restore has not been proven on a separate target path.
- System Owner or QA/Laboratory approval is missing.
